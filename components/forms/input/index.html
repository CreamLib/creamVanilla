      Regular Text Input

      type
      The type of the Input
      id
      Input's general idetnifier
      value
      Input's current value (optional)
      for
      Label attribute. Must be the same of the id
      autocomplete
      Defaults to "on" (optional)
      maxlength
      The maximum allowed value length (optional)
      disabled
      When present, the element is disabled/read-only (optional)
      placeholder
      Placeholder value (optional)

      The id attribute of an input is very important for form accessibility. It allows the user-agent to associate a label to its input.

      Placeholders don't replace labels; they should give hints on how to fill the input.

      Mandatory field (need libraire.js)

      Disabled field (need libraire.js)

                    
              <!-- Regular Input -->
              <p class="ui-input">
                <label for="name1">Enter your name</label>
                <input type="text" id="name1" name="name1" placeholder="a placeholder">
              </p>

              <!-- Mandatory Input -->
              <p class="ui-input">
                <label for="name2">Enter your name</label>
                <input type="text" id="name2" name="name2" placeholder="a placeholder" required>
              </p>

              <!-- Disabled Input -->
              <p class="ui-input">
                <label for="name3">Enter your name</label>
                <type="text" id="name3" name="name3" placeholder="a placeholder" disabled>
              </p>

      Search Input

      The input type search will inherit all attributes from the generic input element.

                                
                          <!-- SEARCH INPUT -->
                          <p class="ui-input search" role="search">
                            <label for="main-search">Search the site</label>
                            <input type="search" id="main-search" name="main-search" autocomplete="on">
                            <button type="submit"><!-- Insert an img or text--></button>
                          </p>
                          
                        
                                  .toto {
                              color: #FFF;
                            }

      Phone Input

      The input type tel will inherit all attributes from the generic input element. In addition, you may use also:

      pattern
      regular expression against which the control's value is to be checked (optional)
      title
      gives description of the pattern (mandatory if pattern is used)

      If you're using the pattern attribute, you should add a title attribute to give a description of the pattern.

      For pattern examples, check HTML Pattern.

                                  
                                <!-- INPUT PHONE -->
                                <p class="ui-input tel">
                                    <label for="phoneTelId">Enter your phone number</label>
                                    <input type="tel" id="phoneTelId" name="phoneTelName" placeholder="+xx x.xx.xx.xx.xx"
                                        title="French number please" pattern="^(?:0|\(?\+33\)?\s?|0033\s?)[1-79](?:[\.\-\s]?\d\d){4}$">
                                </p>
                            
                        
                                  .toto {
                                color: #FFF;
                            }

      E-mail Input

      The input type email will inherit all attributes from the generic input element. In addition, you may use also:

      multiple
      If present, allows the user to enter more than one value (optional)
                                  
                                <!-- INPUT MAIL -->
                                <p class="ui-input mail">
                                    <label for="mailId">Enter your e-mail adress</label>
                                    <input type="email" id="mailId" name="mailName" placeholder="peter-parker@dailybugle.com">
                                </p>

                                color: #FFF;

      Contextual Error Message

      The field is not correct

      Contextual Input Error Message.

      Always place error messages before the associated field, so it can be read by screen-readers before the field presenting the error.

                                  
                                <!-- INPUT ERROR MSG -->
                                <p class="error-msg">
                                    <svg style="width:24px;height:24px" viewBox="0 0 24 24">
                                        <path d="M13,13H11V7H13M13,17H11V15H13M12,2A10,10 0 0,0 2,12A10,10 0 0,0 12,22A10,10 0 0,0 22,12A10,10 0 0,0 12,2Z" />
                                    </svg>
                                    The field is not correct
                                </p>

                                <!-- SIMPLE INPUT -->
                                <p class="ui-input">
                                    <label for="aField">Enter your name</label>
                                    <input type="text" id="aField" name="aField" placeholder="oh, my!">
                                </p>
